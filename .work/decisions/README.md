# .work/decisions/

Architecture decision records (ADRs). Written by the `new-decision` flow.

## Naming

`ADR-NNN-slug.md` — e.g. `ADR-001-state-backend.md`.

## Format

```markdown
# ADR-NNN — Title
Date: YYYY-MM-DD | Status: Accepted | Phase: N
## Decision
One sentence.
## Context
Why this decision was needed.
## Options considered
- option: why rejected or chosen
## Consequences
What this means going forward.
```

## Why kept

The `new-decision` flow writes here; dispatch-rules.json mandates `historian`
review on any change to this directory.
