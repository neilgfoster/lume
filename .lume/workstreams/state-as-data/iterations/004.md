# Iteration 004 - P2: state.json I/O behind the Repository seam

## DoD

- [x] A `state` module defines the per-workstream state.json shape - top-level
      `{workstream, iterations: [], plan: []}` - with a named constant for the
      filename (state.STATE_FILE).
- [x] `state.load(path)` parses state.json and validates every part via P1
      (workstream + each iteration + each plan item); a missing file, malformed
      JSON, or schema failure each raise a clear named error (SchemaError /
      LumeError) identifying the problem.
- [x] `state.save(path, doc)` validates every entity BEFORE writing, then writes
      deterministic diff-friendly JSON (indent=2, stable key order, trailing
      newline). Invalid input never reaches disk (test asserts file absent).
- [x] Round-trip proven: save then load reproduces the document; a second save of
      the loaded doc is byte-identical (idempotent).
- [x] Repository gains `load_state(slug)` / `save_state(slug, doc)` resolving
      `<workstream>/state.json` behind the existing tracking seam.
- [x] Scope held: NO migration (P3) and NO existing verb/model rewiring (P4); no
      markdown files changed. Existing Workstream/Iteration code untouched.
- [x] Tests: state load/save/round-trip/validation-rejection + Repository
      load_state/save_state, against a temp tree. Full suite green (98).
- [x] No dead code / unused imports introduced.

## Self-review
Adversarial pass against the DoD:
- "Invalid never reaches disk" is the load-bearing claim - proven by a test that
  saves a doc with a bad phase and asserts the file does NOT exist afterward, not
  just that save raised. Validation runs before the write, so a partial/corrupt
  file is impossible.
- Validation is enforced on BOTH load and save. A hand-corrupted state.json (bad
  enum, missing key, non-array iterations, malformed JSON) fails loudly with a
  named error at load, never silently downstream - the whole point of the
  workstream.
- Determinism: dumps uses sort_keys + indent + trailing newline, so re-saving a
  loaded doc is byte-identical (tested) - git diffs stay minimal and the file
  doesn't churn on rewrite.
- Scope genuinely held: state.py imported only by repository.py's new methods;
  no existing verb, model, or markdown touched. grep-checked. The mutating verbs
  still run entirely on markdown - that swap is P4.
- Seam fit (decision f): the JSON store is reached through Repository, not by
  callers touching paths, so a future GitHub/Jira backing swaps Repository alone.
- No dead code: every import in state.py and the new repository.py lines is used.
