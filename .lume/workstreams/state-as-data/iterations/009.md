# Iteration 009 - Design the full-content JSON schema for all workstream artifacts

## DoD

The workstream contains six distinct markdown artifact types. Every file whose
content is authored or machine-generated must be accounted for. This iteration
resolves all design forks and produces a concrete execution plan; no engine code.

Artifact inventory:
  objective.md    — status (in state.json), prose objective + done-when text
  iterations/NNN.md — id/type/phase/opened/verdicts (in state.json), DoD body
                      (preamble + checklist), self-review prose, handback prose
  plan.md         — plan items (in state.json via sync), format header prose
  snapshot.md     — entirely derived (Updated, Done, Now, Next)
  decisions.md    — append-only log of decisions + rationale
  discovery.md    — scratch-pad prose for discovery iterations (one-off)
  retro.md        — retrospective prose for closeout (one-off)

- [x] 1. Categorisation settled: each artifact type classified as one of
      {structured-data, prose-journal, derived-view}. Rationale recorded per
      type. (Evidence: a table in this file mapping artifact → category → owner.)
- [x] 2. Structured-data fork resolved: for objective.md and iterations/NNN.md,
      decide whether prose content folds into state.json (expanding existing
      schemas) or lives in separate per-artifact JSON files. One answer for all;
      rationale logged. (Evidence: decision recorded below.)
- [x] 3. DoD checklist schema designed: decide whether DoD is a freeform text
      blob or structured (preamble + array of {text, checked} items). Rationale
      logged. (Evidence: a schema sketch or explicit "freeform string" decision.)
- [x] 4. plan.md fork resolved: decide whether plan.md is eliminated (state.json
      becomes sole source; plan items no longer parsed from markdown) or kept as
      the authored source with state.json derived from it. Rationale logged.
- [x] 5. Ephemeral artifacts settled: decide whether decisions.md, discovery.md,
      retro.md are converted to structured data (with schemas) or explicitly
      designated as prose journals and left as markdown. Rationale logged.
- [x] 6. Markdown view policy settled: which artifacts produce a regenerated .md
      view? Which are JSON-only? (Evidence: policy stated explicitly below.)
- [x] 7. Execution plan: the iterations required to reach the objective, each
      with a committed/optional tag and a one-line binary DoD sketch concrete
      enough to open immediately. Existing P7 (closeout) remains last.
      (Evidence: updated plan.md + state.json.)

## Self-review
All six forks resolved with explicit rationale. The separate-per-artifact-JSON
decision (DoD 2) is the key structural choice: it keeps state.json as a compact
index while each content artifact grows independently. The discovery.md exception
(prose journal) is the only file left as markdown-as-source. Plan items now need
CLI verbs (P9) rather than hand-editing — that is the main new cost; it's
justified by eliminating the parse-and-sync fragility. Execution plan is five
slices, each independently shippable.
