# Iteration 011 - P9: Flip plan — state.json as sole source

## DoD

Retire plan.md as the authored source. State.json `plan` array becomes the sole
source of truth for plan items; plan.md becomes a derived view regenerated by the
engine. Two new CLI verbs let the operator mutate plan items without hand-editing
files.

Code changes are in `workstream.py`, `plan.py`, and `cli.py` only.

- [x] 1. `Workstream.plan_items()` reads from `self._state["plan"]` (not plan.md).
      Returns a list of PlanItem when items exist; None when empty (preserves
      hand-authored Next in snapshot). No `parse_plan` call in workstream.py.
      (Evidence: grep shows no `parse_plan` call remaining in workstream.py.)
- [x] 2. `Workstream._save_state()` no longer syncs plan.md → state["plan"].
      Calls `_render_plan()` after saving state; `record_snapshot()` also calls
      `_render_plan()` so `lume snapshot` regenerates both views.
      (Evidence: grep shows no `parse_plan` call in _save_state.)
- [x] 3. `PlanItem.from_entity(entity: dict) -> PlanItem` added to plan.py.
      Round-trips through `to_entity()` without loss. (Evidence: unit test.)
- [x] 4. `render_plan(items, title) -> str` added to plan.py. Produces one
      `- P<n> | ... | sketch` line per item; iter as zero-padded NNN or `-`.
      (Evidence: rendered output round-trips through parse_plan; unit tests.)
- [x] 5. `lume [-w <slug>] plan add "<sketch>"`: appends a new plan item with
      the next P<n> id, default type `execution`, default tag `committed`.
      Saves state, regenerates plan.md. Exits 0; prints the new item id.
      (Evidence: unit tests.)
- [x] 6. `lume [-w <slug>] plan link <plan-id> <iter-id>`: sets the `iter` field
      of the matching plan item. Plan item not found exits 1. Saves state,
      regenerates plan.md. (Evidence: unit tests.)
- [x] 7. No machine field is parsed from plan.md in normal engine operation.
      `parse_plan` remains only in plan.py (definition) + migrate.py (legacy
      converter) + tests. (Evidence: grep lume/ shows only those locations.)
- [x] 8. All live workstream plan.md files regenerated from state.json plan arrays.
      state-as-data: via `lume snapshot`. lifecycle (closed): direct render call,
      round-trip verified (P1–P6 ids match). build-lume: no plan items; no plan.md.
      (Evidence: plan.md files updated; lifecycle round-trip printed ok.)
- [x] 9. Full suite green; 17 new tests covering from_entity, render_plan
      round-trip, plan_items-from-state, plan add, plan link. Existing stale test
      updated to assert plan.md regenerated from state (not plan.md → state).
      (Evidence: 181 tests, 0 failures.)

## Self-review
Core flip: `plan_items()` reads `state["plan"]`; `_save_state()` stops syncing
from plan.md and instead calls `_render_plan()` after every save. `record_snapshot()`
also calls `_render_plan()` so `lume snapshot` regenerates both views atomically.

`parse_plan` retained in plan.py + migrate.py only — the migration path is the
intended legacy reader and is explicitly called out in the module docstring.

The `plan_items() → None` when empty was the key correctness subtlety: returning
`[]` broke `build_snapshot`'s "preserve hand-authored Next" contract. The fix
(return None for empty) preserves all downstream behaviour while still reading
from state rather than disk.

`lume plan add` uses `-t` (already extracted by `_extract_flag`); `plan link`
takes positional args. No `-g` flag wired for tag — the sketch arg is rest[3],
type comes from the existing `-t` opt. `plan` subcommand dispatched before the
workstream-resolution-dependent block (needs the workstream, so after resolution).

17 new tests; 1 test renamed/rewritten (plan_md_mirrored → plan_md_regenerated).
181 total, all green.
