# Workstream Lifecycle

Refine the workstream process into an explicit, phased lifecycle (the
"practice contract"): a workstream runs as phased practice rather than a
flat iteration list -

  discovery (build context) -> planning (plan the phases to hit the
  objective) -> execution iterations (run to completion) -> close-out.

Formalises what build-lume did ad hoc and exercises the practice seam,
which build-lume never built.

Done-when: a workstream runs through typed/phased iterations (not a flat
list); the tooling tracks active vs closed workstreams and which one is
current (so >1 workstream can coexist); and this workstream itself is run
through that lifecycle to prove it.
